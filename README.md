# OCD Screening — Clinical Decision Support

A machine-learning web app, framed as a **clinician decision-support / triage aid**,
that reads a patient's free-text intake and estimates which mental-health condition
it most resembles (**OCD, depression, ADHD, or PTSD**), shows the language driving
that call, abstains when unsure, and surfaces **evidence-based treatment options**
for the clinician to review. It is built on **real Reddit mental-health posts** and
is an end-to-end ML pipeline: data → text classifier → screening decision (with
abstention) → explanation → guidance → deployed web app.

> This is **not** a diagnostic tool. Screening is text-similarity, can be wrong, and
> is designed to assist a licensed clinician who reviews every output and makes all
> clinical decisions. Do not enter identifying patient information.

## Results (held-out test set)

Trained on ~20k real posts across 4 conditions (TF-IDF + Logistic Regression):

| Metric | Score |
|---|---|
| Majority-class baseline | 0.296 |
| 5-fold CV accuracy | 0.837 ± 0.006 |
| **Held-out test accuracy** | **0.843** |
| Held-out test macro-F1 | 0.826 |
| **OCD class** | precision **0.90**, recall **0.86**, F1 **0.88** |

Model comparison (same TF-IDF features, `reports/model_comparison.txt`):

| Model | Test accuracy | Test macro-F1 |
|---|---|---|
| Dummy (most frequent) | 0.295 | 0.114 |
| **Logistic Regression** (used) | **0.843** | **0.826** |
| Linear SVM | 0.834 | 0.818 |

Logistic Regression is used because it is competitive *and* exposes probabilities
(for the confidence threshold) and coefficients (for explanations). Full report,
confusion matrix, and top words are in `reports/`. The model also yields highly
interpretable signals — e.g. the top OCD terms are *intrusive, compulsions, erp,
obsessions, rituals, reassurance*.

## Features

- **Multi-condition screening** with probability bars for each condition.
- **Per-prediction explanations** — highlights the exact words that drove the
  prediction (TF-IDF × model weight), i.e. feature attribution.
- **Confidence threshold** — if the top probability is below `CONFIDENCE_THRESHOLD`
  the app reports *"Uncertain / not enough signal"* and abstains instead of guessing
  (the screening analogue of novelty/anomaly detection).
- **Evidence-based guidance** for the predicted condition (first-line meds, other
  options, and therapies), generated by an LLM with a built-in fallback.

## How to run

1. Create an environment and install dependencies (Python 3.10–3.12 recommended):
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

2. (Optional) Retrain from scratch:
   ```bash
   python download_data.py   # pulls real Reddit data into data/
   python train.py           # writes models/ and reports/
   ```
   A pre-trained model is already included in `models/`, so you can skip this.

3. Run the web app:
   ```bash
   python app.py
   ```
   Open http://localhost:5001 and paste a description to screen.

### Optional live AI guidance layer (free or paid)

By default the app shows built-in, evidence-based guidance (no API key needed).
To have an LLM phrase the guidance for the specific input, set environment
variables (locally in `key.env`, or in your host's dashboard). The provider is
configurable via an OpenAI-compatible API, so you can use a **free** option:

| Variable | Free (Groq) | Free (Gemini) | Paid (OpenAI) |
|---|---|---|---|
| `LLM_API_KEY` | your Groq key | your Gemini key | your OpenAI key |
| `LLM_BASE_URL` | `https://api.groq.com/openai/v1` | `https://generativelanguage.googleapis.com/v1beta/openai/` | *(leave unset)* |
| `LLM_MODEL` | `llama-3.3-70b-versatile` | `gemini-2.0-flash` | `gpt-4o-mini` |

`OPENAI_API_KEY` is still accepted as a fallback for `LLM_API_KEY`. Because the
call runs server-side, it works even on networks that block these domains locally.

## Project files

- **`download_data.py`** – downloads the real Reddit dataset (Hugging Face, public domain).
- **`train.py`** – cleans text, trains the TF-IDF + Logistic Regression classifier,
  and writes evaluation metrics, a confusion matrix, and top predictive words to `reports/`.
- **`app.py`** – Flask web app: text in → predicted condition + probabilities →
  evidence-based (or GPT-phrased) guidance, with a clinician-approval disclaimer.
- **`models/`** – the saved trained pipeline.
- **`reports/`** – metrics, confusion matrix, and interpretability output.
